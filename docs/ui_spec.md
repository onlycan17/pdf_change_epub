# UI/UX 명세서 (v3 - Desktop Optimized)

## 1. 개요
이 문서는 PDF → EPUB 변환기의 웹 인터페이스 디자인과 사용자 경험(UX)에 대한 요구사항을 정의합니다. 데스크탑 브라우저 환경에 최적화된, 시각적으로 아름답고 직관적인 UI를 설계하는 것을 목표로 합니다.

## 2. 디자인 원칙
- **직관성**: 사용자가 쉽게 이해하고 사용할 수 있어야 함.
- **간결성**: 불필요한 요소를 최소화하여 핵심 기능에 집중.
- **데스크탑 최적화**: 1280px 이상의 넓은 화면에 최적화된 고정 레이아웃을 제공.
- **심미적 만족감**: 모던한 디자인 요소(그림자, 그라데이션, 부드러운 전환 효과)를 활용하여 시각적으로 즐거운 경험을 제공.
- **일관성**: 전체 애플리케이션에서 일관된 디자인 언어 유지.

## 3. 색상 시스템
- **주요 색상**:
  - Primary: #4F46E5 (인디고)
  - Secondary: #10B981 (에메랄드 그린)
  - Background: #F9FAFB (밝은 회색)
  - Surface: #FFFFFF (흰색)
  - Text: #111827 (어두운 회색)
- **상태 색상**:
  - Success: #10B981 (녹색)
  - Warning: #F59E0B (주황색)
  - Error: #EF4444 (빨간색)
  - Info: #3B82F6 (파란색)

## 4. 타이포그래피
- **폰트**: 'Pretendard' (한국어 최적화), 대체 폰트로 'Noto Sans KR'
- **폰트 크기**:
  - Heading 1: 32px, Bold
  - Heading 2: 24px, SemiBold
  - Heading 3: 20px, SemiBold
  - Body Large: 18px, Regular
  - Body Medium: 16px, Regular
  - Body Small: 14px, Regular
  - Caption: 12px, Regular

## 5. 컴포넌트 설계

### 5.1. 헤더
- 애플리케이션 로고 및 이름
- 로그인 / 회원가입 버튼 (로그아웃 상태 시)
- 내 정보 / 로그아웃 버튼 (로그인 상태 시)
- 요금제 안내 버튼
- 다크 모드 토글 버튼
- 언어 선택 드롭다운 (한국어/영어)

### 5.2. 파일 업로드 영역
- 드래그 앤 드롭 영역 및 파일 선택 버튼.
- 안내 문구: "최대 20MB까지 업로드할 수 있습니다." (무료 사용자 기준) / "Pro 등급 사용자는 최대 200MB까지 업로드할 수 있습니다." (유료 사용자 기준)
- 업로드된 파일 정보 표시 (파일명, 크기).
- 파일 제거 기능.

### 5.3. 설정 패널
- **OCR 사용 여부 선택**:
    - 시스템이 스캔된 PDF라고 판단하면, "OCR을 사용하여 텍스트를 추출하시겠습니까?" 체크박스를 활성화하여 제시.
    - 사용자가 직접 선택 또는 해제 가능.
- **OCR 언어 선택**: (기존과 동일)
- **EPUB 메타데이터 입력 필드**: (기존과 동일)

### 5.4. 진행 상황 표시기
- 전체 진행률 바
- 단계별 상태 표시 (PDF 분석, 텍스트 보정, EPUB 생성 등)
- 각 단계의 상세 정보 및 예상 시간

### 5.5. 결과 표시 영역
- 변환 완료 메시지
- EPUB 파일 다운로드 버튼
- 변환된 파일 정보 (페이지 수, 이미지 수 등)
- 새 파일 변환 버튼

### 5.6. 에러 표시 영역
- 에러 메시지 표시
- 에러 해결을 위한 가이드
- 재시도 버튼

## 6. 페이지 구조

### 6.1. 홈페이지
- 애플리케이션 소개 및 주요 기능 설명
- 파일 업로드 영역
- 최근 변환 기록 (로그인 기능 구현 시)

### 6.2. 변환 진행 페이지
- 진행 상황 표시기
- 설정 패널
- 실시간 로그 출력 영역

### 6.3. 결과 페이지
- 변환 결과 요약
- 다운로드 옵션
- 소셜 공유 기능

### 6.4. 요금제 페이지 (신규)
- **목표**: 사용자가 각 플랜(무료, 단일 결제, 구독)의 특징과 가격을 명확히 비교하고 결제를 진행할 수 있도록 안내.
- **구성 요소**:
    - 등급별 기능 비교표 (파일 크기, 변환 횟수, LLM 보정 품질 등).
    - 단일 결제 및 구독 플랜 선택 버튼.
    - 결제 모듈 연동 (Stripe/PortOne).

## 7. 레이아웃 (Layout)
애플리케이션은 데스크탑 브라우저 환경에 최적화된 고정 너비(최대 1440px)의 3단 레이아웃을 사용합니다.

- **왼쪽 패널 (20%)**:
    - 최근 변환 작업 목록 (로그인 시).
    - 서비스 네비게이션 메뉴.
- **중앙 패널 (50%)**:
    - 핵심 상호작용 영역.
    - 파일 업로드, 변환 설정, 진행 상태, 결과 다운로드가 순차적으로 이 영역에 표시됨.
- **오른쪽 패널 (30%)**:
    - 상세 정보 및 미리보기 영역.
    - 업로드된 PDF의 정보(페이지 수, 파일 크기) 또는 1페이지 미리보기.
    - 변환 중에는 상세 로그 표시.

## 12. 국제화(i18n)
애플리케이션은 글로벌 사용자를 위해 한국어와 영어를 모두 지원합니다.

- **UI 언어 지원**:
  - 기본 언어는 사용자의 브라우저 설정을 따릅니다.
  - 헤더의 언어 선택 드롭다운을 통해 사용자가 언제든지 UI 언어를 한국어(KO) 또는 영어(EN)로 명시적으로 변경할 수 있습니다.
  - 모든 UI 텍스트(버튼, 레이블, 메시지 등)는 선택된 언어에 맞게 표시됩니다.

- **콘텐츠 언어 지원 (OCR)**:
  - 사용자는 '설정 패널'에서 변환할 PDF 문서의 주요 언어(한국어/영어)를 선택할 수 있습니다.
  - 이 선택은 백엔드 OCR 처리 시 정확도를 높이는 데 사용됩니다.
  - 기본값은 '한국어'로 설정합니다.
