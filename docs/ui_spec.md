# UI/UX 명세서 (v4 - 상세 설계)

## 1. 개요

### 1.1. 문서 목적
이 문서는 PDF → EPUB 변환기의 웹 인터페이스 디자인과 사용자 경험(UX)에 대한 상세한 요구사항을 정의합니다. 데스크탑 브라우저 환경에 최적화된, 시각적으로 아름답고 직관적인 UI를 설계하는 것을 목표로 하며, 개발팀과 디자인팀이 일관된 방향성을 유지하며 구현할 수 있도록 구체적인 가이드라인을 제공합니다.

### 1.2. 타겟 사용자
- **주요 사용자**: 전자책을 선호하는 일반 사용자, 학술 연구자, 문서 관리자
- **기술 수준**: 중간 수준의 컴퓨터 사용 능력을 가진 사용자
- **사용 환경**: 주로 데스크탑 환경에서 사용, 최소 1280px 이상의 해상도 지원

### 1.3. 핵심 사용자 시나리오
1. **빠른 변환**: 사용자가 PDF 파일을 업로드하고 최소한의 설정으로 빠르게 EPUB로 변환
2. **고급 변환**: OCR 설정, 메타데이터 입력 등 상세한 옵션을 통한 맞춤 변환
3. **대량 처리**: 여러 PDF 파일을 일괄 변환하는 사용자 시나리오
4. **품질 관리**: 변환 결과를 미리보기하고 품질을 확인한 후 다운로드

## 2. 디자인 원칙

### 2.1. 핵심 원칙
- **직관성**: 사용자가 별도의 학습 없이 즉시 사용할 수 있어야 함. 복잡한 기능은 단계별로 노출
- **간결성**: 불필요한 요소를 최소화하여 핵심 기능에 집중. 정보 과부하 방지
- **데스크탑 최적화**: 1280px 이상의 넓은 화면에 최적화된 고정 레이아웃을 제공. 1440px를 기준으로 설계
- **심미적 만족감**: 모던한 디자인 요소(그림자, 그라데이션, 부드러운 전환 효과)를 활용하여 시각적으로 즐거운 경험을 제공
- **일관성**: 전체 애플리케이션에서 일관된 디자인 언어 유지. 컴포넌트 재사용성 극대화

### 2.2. 사용자 경험 원칙
- **피드백 즉시성**: 모든 사용자 행동에 대한 즉각적인 시각적 피드백 제공
- **오류 방지**: 사용자가 실수를 할 수 없는 인터페이스 설계 (예: 잘못된 파일 형식 업로드 방지)
- **복구 가능성**: 오류 발생 시 쉽게 이전 상태로 돌아갈 수 있는 경로 제공
- **진행 상황 투명성**: 긴 처리 시간이 필요한 작업의 경우 상세한 진행 상황을 시각적으로 표시

## 3. 색상 시스템

### 3.1. 주요 색상 팔레트
- **Primary**: #4F46E5 (인디고)
  - 용도: 주요 버튼, 활성 상태, 강조 요소
  - 접근성 대비: 흰색 텍스트 대비 4.8:1 (AA 준수)
- **Secondary**: #10B981 (에메랄드 그린)
  - 용도: 성공 상태, 보조 버튼, 긍정적 피드백
  - 접근성 대비: 흰색 텍스트 대비 3.3:1 (AA 준수)
- **Background**: #F9FAFB (밝은 회색)
  - 용도: 전체 배경, 카드 영역 배경
- **Surface**: #FFFFFF (흰색)
  - 용도: 카드, 모달, 패널 등 요소 배경
- **Text Primary**: #111827 (어두운 회색)
  - 용도: 주요 텍스트, 제목
- **Text Secondary**: #6B7280 (중간 회색)
  - 용도: 부가 설명, 레이블

### 3.2. 상태 색상
- **Success**: #10B981 (녹색)
  - 용도: 성공 메시지, 완료 상태, 긍정적 피드백
  - 대체 색상: #059669 (더 어두운 녹색)
- **Warning**: #F59E0B (주황색)
  - 용도: 경고 메시지, 주의 필요 상태
  - 대체 색상: #D97706 (더 어두운 주황색)
- **Error**: #EF4444 (빨간색)
  - 용도: 오류 메시지, 실패 상태, 부정적 피드백
  - 대체 색상: #DC2626 (더 어두운 빨간색)
- **Info**: #3B82F6 (파란색)
  - 용도: 정보 메시지, 안내 텍스트
  - 대체 색상: #2563EB (더 어두운 파란색)

### 3.3. 다크 모드 색상
- **Dark Background**: #111827 (매우 어두운 회색)
- **Dark Surface**: #1F2937 (어두운 회색)
- **Dark Surface Elevated**: #374151 (중간 어두운 회색)
- **Dark Text Primary**: #F9FAFB (밝은 회색)
- **Dark Text Secondary**: #D1D5DB (중간 밝은 회색)

### 3.4. 색상 사용 가이드라인
- Primary 색상은 전체 화면의 10% 이내로 사용하여 시각적 피로도 감소
- 상태 색상은 해당 상황에서만 사용하여 색상의 의미 유지
- 텍스트와 배경의 대비비는 최소 4.5:1 유지 (WCAG 2.1 AA 준수)
- 색상만으로 정보를 전달하지 않고 아이콘이나 텍스트와 함께 사용

## 4. 타이포그래피

### 4.1. 폰트 시스템
- **주요 폰트**: 'Pretendard' (한국어 최적화)
- **대체 폰트**: 'Noto Sans KR' ( Pretendard 로딩 실패 시)
- **영문 폰트**: 'Inter' (숫자 및 영문 표현 최적화)
- **모노스페이스 폰트**: 'JetBrains Mono' (코드, 파일명 등 기술 정보 표시)

### 4.2. 폰트 크기 및 굵기 시스템

| Display Medium | 40px | Bold | 1.2 | 섹션 메인 타이틀 |
| Display Small | 32px | Bold | 1.2 | 페이지 메인 타이틀 |
| Heading Large | 24px | SemiBold | 1.3 | 주요 섹션 헤딩 |
| Heading Medium | 20px | SemiBold | 1.3 | 서브 섹션 헤딩 |
| Heading Small | 18px | SemiBold | 1.4 | 카드 타이틀, 소제목 |
| Body Large | 18px | Regular | 1.5 | 본문, 중요 설명 |
| Body Medium | 16px | Regular | 1.5 | 기본 본문, 레이블 |
| Body Small | 14px | Regular | 1.5 | 부가 정보, 캡션 |
| Caption | 12px | Regular | 1.4 | 작은 주석, 메타데이터 |

### 4.3. 타이포그래피 사용 가이드라인
- 한 페이지 내에서 최대 3종류의 폰트 크기만 사용하여 시각적 계층 구조 유지
- 굵기는 Regular, SemiBold, Bold 세 가지만 사용하여 일관성 유지
- 행간은 폰트 크기의 1.4배 ~ 1.6배 사이로 설정하여 가독성 확보
- 텍스트 블록의 최대 너비는 70~80자로 제한하여 가독성 향상

## 5. 컴포넌트 설계

### 5.1. 헤더 (Header)

#### 5.1.1. 구성 요소
- **로고 영역**:
  - 애플리케이션 로고 (SVG, 40x40px)
  - 애플리케이션 이름 "PDF to EPUB" (Heading Small, SemiBold)
  - 클릭 시 홈페이지로 이동
- **네비게이션 메뉴**:
  - 홈, 기능 소개, 요금제, 문의 (로그인 상태无关)
  - 현재 페이지 표시 (Primary 색상의 밑줄)
  - 호버 시 배경색 변화 (#F3F4F6)
- **사용자 메뉴**:
  - 로그인 상태无关:
    - 로그인 버튼 (Secondary, Medium)
    - 회원가입 버튼 (Primary, Medium)
  - 로그인 상태:
    - 사용자 프로필 이미지 (32x32px, 원형)
    - 사용자 이름 표시 (Body Medium)
    - 드롭다운 메뉴: 내 정보, 설정, 로그아웃
- **기능 버튼**:
  - 요금제 안내 버튼 (Outline, Small)
  - 다크 모드 토글 버튼 (Icon Only, 24x24px)
  - 언어 선택 드롭다운 (한국어/영어, Small)

#### 5.1.2. 상태별 디자인
- **기본 상태**: 배경색 Surface, 그림자 0px 1px 3px rgba(0, 0, 0, 0.1)
- **스크롤 시**: 배경색 불투명도 95%로 변경, 그림자 0px 4px 6px rgba(0, 0, 0, 0.1)
- **다크 모드**: 배경색 Dark Surface, 그림자 0px 1px 3px rgba(0, 0, 0, 0.3)

#### 5.1.3. 반응형 동작
- 1280px 미만: 사용자 메뉴 간소화, 일부 메뉴 아이콘화
- 768px 미만: 모바일 메뉴로 전환 (햄버거 메뉴)

### 5.2. 파일 업로드 영역 (File Upload Area)

#### 5.2.1. 구성 요소
- **드래그 앤 드롭 영역**:
  - 크기: 600x400px (최소), 둥근 모서리 16px
  - 테두리: 2px dashed #D1D5DB (기본 상태)
  - 배경색: #F9FAFB
  - 아이콘: Cloud Upload 아이콘 (48x48px, #9CA3AF)
  - 제목: "PDF 파일을 여기에 드래그하거나 클릭하여 선택하세요" (Body Large)
  - 설명: "최대 20MB까지 업로드할 수 있습니다." (Body Small, Text Secondary)
- **파일 선택 버튼**:
  - 스타일: Primary, Medium
  - 텍스트: "파일 선택"
  - 아이콘: Folder Open 아이콘 (왼쪽)
- **업로드된 파일 정보**:
  - 파일 아이콘: File Text 아이콘 (24x24px, #6B7280)
  - 파일명: Body Medium (최대 30자, 초과 시 ...으로 표시)
  - 파일 크기: Body Small, Text Secondary
  - 제거 버튼: X 아이콘 (16x16px, #6B7280), 호버 시 #EF4444

#### 5.2.2. 상태별 디자인
- **기본 상태**: 테두리 #D1D5DB, 배경색 #F9FAFB
- **드래그 오버 상태**: 테두리 Primary 색상, 배경색 #EEF2FF
- **파일 선택 완료 상태**: 테두리 #10B981, 배경색 #ECFDF5
- **업로드 진행 상태**: 프로그레스 바 표시, 파일 정보 영역으로 전환
- **에러 상태**: 테두리 #EF4444, 배경색 #FEF2F2, 에러 메시지 표시

#### 5.2.3. 사용자 등급별 차이
- **무료 사용자**:
  - 설명: "최대 20MB까지 업로드할 수 있습니다."
  - 업로드 가능한 파일 수: 1개
  - 업로드 횟수 제한: 일일 5회
- **프리미엄 사용자**:
  - 설명: "Pro 등급 사용자는 최대 200MB까지 업로드할 수 있습니다."
  - 업로드 가능한 파일 수: 5개 (일괄 처리)
  - 업로드 횟수 제한: 무제한

### 5.3. 설정 패널 (Settings Panel)

#### 5.3.1. 구성 요소
- **OCR 설정 영역**:
  - OCR 사용 여부 체크박스 (기본값: 비활성화)
  - OCR 활성화 조건: 시스템이 스캔된 PDF라고 판단할 때 자동으로 활성화 제안
  - OCR 설명 텍스트: "스캔된 PDF 문서의 텍스트를 추출합니다. 처리 시간이 길어질 수 있습니다." (Body Small)
- **OCR 언어 선택**:
  - 드롭다운 메뉴: 한국어, 영어, 한국어+영어
  - 기본값: 한국어
  - 아이콘: Languages 아이콘 (오른쪽)
- **EPUB 메타데이터 입력 필드**:
  - 제목 입력 필드 (필수, 최대 100자)
  - 저자 입력 필드 (선택, 최대 50자)
  - 설명 입력 필드 (선택, 최대 500자, textarea)
  - 표지 이미지 업로드 (선택, 최대 5MB, JPG/PNG)
- **고급 설정**:
  - 접기/펼치기 버튼 (기본값: 접힌 상태)
  - 챕터 분할 방식 선택: 페이지별, 챕터 마커 기준, 자동 감지
  - 이미지 품질 선택: 저품질(작은 파일 크기), 보통, 고품질(큰 파일 크기)
  - 폰트 임베딩 여부 체크박스 (기본값: 활성화)

#### 5.3.2. 상태별 디자인
- **기본 상태**: 배경색 Surface, 둥근 모서리 12px, 그림자 0px 1px 3px rgba(0, 0, 0, 0.1)
- **활성 상태**: Primary 색상의 테두리 1px
- **에러 상태**: Error 색상의 테두리 1px, 해당 필드 아래에 에러 메시지 표시

#### 5.3.3. 상호작용
- 필드에 포커스 시 부드러운 확장 애니메이션
- 입력 내용 실시간 유효성 검사
- 고급 설정 영역은 접기/펼치기 애니메이션 적용

### 5.4. 진행 상황 표시기 (Progress Tracker)

#### 5.4.1. 구성 요소
- **전체 진행률 바**:
  - 너비: 100%, 높이: 8px
  - 배경색: #E5E7EB
  - 진행 색상: Primary (#4F46E5)
  - 진행률 텍스트: 오른쪽 상단에 표시 (Body Medium)
- **단계별 상태 표시**:
  - 단계 목록: 파일 분석 → 텍스트 추출 → 텍스트 보정 → EPUB 생성 → 최종화
  - 각 단계 아이콘: 원형 아이콘 (24x24px)
  - 단계 상태: 대기(회색), 진행 중(Primary 색상, 회전 애니메이션), 완료(체크 아이콘, 녹색), 에러(X 아이콘, 빨간색)
  - 단계 이름: 아이콘 오른쪽에 표시 (Body Small)
- **상세 정보 영역**:
  - 현재 단계 상세 설명 (Body Medium)
  - 예상 시간: "약 3분 남음" (Body Small, Text Secondary)
  - 처리 중인 파일명 표시
  - 실시간 로그 표시 영역 (최대 5줄, 스크롤 가능)

#### 5.4.2. 상태별 디자인
- **대기 상태**: 모든 단계 아이콘이 회색, 진행률 0%
- **진행 중 상태**: 현재 단계 아이콘이 Primary 색상으로 채워지고 회전 애니메이션, 이전 단계는 체크 표시
- **완료 상태**: 모든 단계 아이콘이 체크 표시, 진행률 100%
- **에러 상태**: 해당 단계 아이콘이 빨간색 X 표시, 진행률 바 중단

#### 5.4.3. 애니메이션
- 진행률 바: 부드러운 너비 변화 애니메이션 (0.3초)
- 단계 전환: 페이드 인/아웃 효과 (0.5초)
- 로그 표시: 새 로그 항목이 아래에서 부드럽게 추가되는 애니메이션

### 5.5. 결과 표시 영역 (Result Display)

#### 7.2.3. 오른쪽 패널 (30%)
- **너비**: 432px (1440px 기준)
- **구성 요소**:
  - 상세 정보 및 미리보기 영역:
    - 영역 제목: "정보" (Body Small, SemiBold)

      - 파일명 (Body Medium)

    - 제목: "도움말" (Body Small, SemiBold)

## 9. 접근성 (Accessibility)

### 9.1. WCAG 2.1 AA 준수 기준

#### 9.1.1. 시각적 접근성
- **색상 대비**:
  - 일반 텍스트: 최소 4.5:1 대비율
  - 큰 텍스트(18px 이상 또는 14px 이상 굵게): 최소 3:1 대비율
  - 비텍스트 요소: 최소 3:1 대비율
- **색상 의존성 최소화**:
  - 색상만으로 정보를 전달하지 않음
  - 아이콘, 텍스트, 패턴 등 추가적인 시각적 단서 제공
- **시각적 구분**:
  - 링크는 밑줄이나 색상으로 일반 텍스트와 구분
  - 포커스 상태가 명확하게 표시됨
  - 비활성화된 요소는 시각적으로 구분됨

#### 9.1.2. 운영적 접근성
- **키보드 내비게이션**:
  - 모든 기능을 키보드만으로 사용 가능
  - 논리적인 탭 순서 유지
  - 포커스 트랩이 있는 모달 및 드롭다운
  - 시각적 포커스 표시 (outline, 배경색 변화 등)
- **단축키**:
  - 주요 기능에 대한 키보드 단축키 제공
  - 단축키 목록은 설정 페이지에서 확인 가능
  - 예: Ctrl/Cmd + U (파일 업로드), Ctrl/Cmd + S (설정)

#### 9.1.3. 인지적 접근성
- **명확한 안내**:
  - 모든 상호작용 요소에 명확한 레이블 제공
  - 에러 메시지는 구체적이고 해결 방법을 포함
  - 복잡한 프로세스는 단계별로 분리
- **일관된 디자인**:
  - 유사한 기능은 유사한 방식으로 작동
  - 전체 애플리케이션에서 일관된 용어 사용
  - 예측 가능한 상호작용 패턴

### 9.2. 스크린 리더 호환성

#### 9.2.1. ARIA 레이블
- **역할 속성**:
  - 모든 대화형 요소에 적절한 role 속성 제공
  - 예: role="button", role="navigation", role="main"
- **이름 속성**:
  - aria-label을 사용하여 요소의 목적을 명확히 설명
  - 아이콘만 있는 버튼에 텍스트 설명 제공
  - 예: aria-label="설정"
- **상태 속성**:
  - aria-expanded, aria-checked, aria-selected 등으로 상태 표시
  - 동적으로 변경되는 상태는 즉시 업데이트

#### 9.2.2. 라이브 리전
- **중요한 변경 사항 알림**:
  - aria-live="polite"를 사용하여 중요하지 않은 변경 사항 알림
  - aria-live="assertive"를 사용하여 중요한 변경 사항 알림
  - 예: 변환 완료, 에러 발생 등
- **상태 업데이트**:
  - 진행 상황 변경 시 aria-live로 업데이트
  - 폼 유효성 검사 결과를 aria-live로 알림

#### 9.2.3. 키보드 내비게이션
- **탭 순서**:
  - 논리적인 탭 순서 유지 (위에서 아래, 왼쪽에서 오른쪽)
  - 시각적으로 보이는 순서와 일치
  - 소스 코드 순서와 일치
- **포커스 관리**:
  - 모달이 열릴 때 포커스를 모달로 이동
  - 모달이 닫힐 때 포커스를 원래 위치로 복원
  - 드롭다운이 열릴 때 포커스를 드롭다운으로 이동

### 9.3. 모바일 접근성

#### 9.3.1. 터치 대상
- **최소 크기**: 44x44px 이상의 터치 영역
- **간격**: 인접한 터치 대상 사이에 최소 8px 간격
- **터치 피드백**: 터치 시 시각적/촉각적 피드백 제공

#### 9.3.2. 반응형 디자인
- **가로 모드**: 모든 기능이 가로 모드에서도 정상 작동
- **확대/축소**: 콘텐츠가 200% 확대되어도 가독성 유지
- **텍스트 크기 조정**: 사용자가 텍스트 크기를 200%까지 조정 가능

### 9.4. 접근성 테스트 방법

#### 9.4.1. 자동화된 테스트
- **도구**: axe-core, WAVE, Lighthouse 접근성 감사
- **테스트 항목**:
  - 색상 대비율
  - ARIA 속성 누락
  - 이미지 대체 텍스트 누락
  - 폼 레이블 누락

#### 9.4.2. 수동 테스트
- **키보드만 사용**: 모든 기능을 키보드만으로 테스트
- **스크린 리더 테스트**: NVDA, VoiceOver, JAWS로 테스트

## 10. 애니메이션 및 전환 효과

### 10.1. 애니메이션 원칙

#### 10.1.1. 목적성
- 모든 애니메이션은 명확한 목적을 가져야 함
- 사용자의 주의를 끌거나 상태 변화를 알리는 데 사용
- 단순한 장식용 애니메이션은 최소화

#### 10.1.2. 성능
- 하드웨어 가속을 사용하는 CSS 속성으로 애니메이션 구현
- transform과 opacity 속성을 주로 사용
- layout, paint를 유발하는 속성은 피함

#### 10.1.3. 접근성
- `prefers-reduced-motion` 미디어 쿼리를 사용하여 애니메이션 제어
- 사용자가 애니메이션을 줄이도록 설정한 경우 애니메이션 비활성화
- 필수적인 피드백은 간단한 방식으로 유지

### 10.2. 페이지 전환 애니메이션

#### 10.2.1. 페이지 로딩
- **페이드 인**: 페이지 콘텐츠가 부드럽게 나타남 (0.5초)
- **슬라이드 업**: 페이지 하단 요소가 위로 슬라이드 (0.7초)
- **스태거**: 여러 요소가 순차적으로 나타남 (0.1초 간격)

#### 10.2.2. 페이지 전환
- **페이드 아웃/인**: 현재 페이지가 사라지고 새 페이지가 나타남 (0.3초)
- **슬라이드**: 페이지가 좌우로 슬라이드 (0.4초)
- **공유 요소 전환**: 두 페이지에 공통된 요소는 부드럽게 이동

### 10.3. 컴포넌트 애니메이션

#### 10.3.1. 버튼
- **호버**: 배경색과 그림자가 부드럽게 변화 (0.2초)
- **클릭**: 크기가 95%로 작아졌다가 복원 (0.1초)
- **포커스**: 포커스 링이 부드럽게 나타남 (0.2초)

#### 10.3.2. 카드
- **호버**: 그림자가 커지고 약간 위로 올라감 (0.3초)
- **나타남**: 아래에서 위로 슬라이드하며 나타남 (0.5초)
- **사라짐**: 위로 슬라이드하며 사라짐 (0.3초)

#### 10.3.3. 모달 및 드롭다운
- **나타남**: 중앙에서 확대되며 나타남 (0.3초)
- **배경**: 배경이 어두워지며 나타남 (0.3초)
- **사라짐**: 중앙으로 축소되며 사라짐 (0.2초)

### 10.4. 상태 변화 애니메이션

#### 10.4.1. 진행 상황 표시
- **프로그레스 바**: 너비가 부드럽게 증가 (0.3초)
- **단계 전환**: 이전 단계가 축소되고 다음 단계가 확대됨 (0.4초)
- **완료 아이콘**: 체크 모양으로 부드럽게 변화 (0.5초)

#### 10.4.2. 로딩 상태
- **스피너**: 부드러운 회전 애니메이션 (1초에 한 바퀴)
- **스켈레톤**: 콘텐츠가 로딩되는 것처럼 보이는 플레이스홀더 (1.5초)
- **펄스**: 주기적으로 밝기가 변하는 효과 (2초 주기)

#### 10.4.3. 에러 상태
- **흔들림**: 좌우로 작게 흔들리는 효과 (0.5초)
- **깜빡임**: 빨간색으로 깜빡이는 효과 (0.5초)
- **확대**: 에러 메시지가 부드럽게 확대되며 나타남 (0.3초)

### 10.5. 마이크로 인터랙션

#### 10.5.1. 체크박스 및 라디오 버튼
- **선택**: 체크 표시가 부드럽게 그려짐 (0.2초)
- **호버**: 배경이 부드럽게 변화 (0.2초)

#### 10.5.2. 토글 스위치
- **전환**: 스위치가 부드럽게 이동 (0.3초)
- **배경색**: 배경색이 부드럽게 변화 (0.3초)

#### 10.5.3. 입력 필드
- **포커스**: 하단 테두리가 Primary 색상으로 변화 (0.2초)
- **입력**: 입력 내용이 부드럽게 나타남 (0.1초)

### 10.6. 애니메이션 타이밍

#### 10.6.1. 이징 함수
- **ease-out**: 빠르게 시작하고 천천히 끝남 (가장 일반적)
- **ease-in-out**: 천천히 시작하고 천천히 끝남 (중요한 전환)
- **linear**: 일정한 속도 (로딩 애니메이션)

#### 10.6.2. 지연 시간

## 12. 국제화 (i18n)

### 12.1. 다국어 지원 전략

#### 12.1.1. 지원 언어
- **한국어 (ko-KR)**: 기본 언어
- **영어 (en-US)**: 주요 지원 언어
- **확장 가능성**: 추가 언어 지원을 위한 구조

#### 12.1.2. 언어 감지 및 전환
- **자동 감지**: 사용자의 브라우저 언어 설정을 기반으로 자동 감지
- **수동 전환**: 헤더의 언어 선택 드롭다운을 통해 수동 전환 가능
- **지속성**: 사용자의 언어 선택은 로컬 스토리지에 저장
- **URL 기반**: 언어별 URL 경로 지원 (예: /ko/home, /en/home)

### 12.2. 텍스트 번역 관리

#### 12.2.1. 번역 파일 구조
```
/locales
  /ko-KR
    common.json
    home.json
    conversion.json

    errors.json
  /en-US
    common.json
    home.json
    conversion.json
    pricing.json
    errors.json
```

#### 12.2.2. 번역 키 명명 규칙
- **계층적 구조**: 페이지.기능.세부기능 형식의 키 사용
- **의미 있는 이름**: 키는 의미를 명확히 전달해야 함
- **일관성**: 유사한 기능은 유사한 키 구조 사용
- **예시**:
  ```json
  {
    "home": {
      "hero": {
        "title": "PDF를 EPUB로 쉽게 변환하세요",

      },

      }
    }
  }
  ```

#### 12.2.3. 동적 텍스트 처리
- **변수 삽입**: "파일 {filename}을(를) 업로드했습니다." 형식의 동적 텍스트 지원
- **복수형 처리**: 언어별 복수형 규칙에 따른 텍스트 처리
- **성별 처리**: 필요한 경우 성별에 따른 텍스트 처리
- **예시**:
  ```json
  {
    "file": {

    }
  }
  ```

### 12.3. 날짜 및 시간 형식

#### 12.3.1. 날짜 형식
- **한국어**: YYYY년 MM월 DD일
- **영어**: MM/DD/YYYY
- **짧은 형식**: YY.MM.DD (한국어), MM/DD/YY (영어)
- **긴 형식**: YYYY년 MM월 DD일 dddd (한국어), dddd, MMMM DD, YYYY (영어)

#### 12.3.2. 시간 형식
- **한국어**: HH시 MM분 SS초 (24시간제)
- **영어**: HH:MM:SS (24시간제) 또는 hh:MM:SS AM/PM (12시간제)
- **상대적 시간**: "n분 전", "n시간 전" 등 상대적 시간 표현

#### 12.3.3. 숫자 형식
- **한국어**: 1,000,000 (쉼표 구분)
- **영어**: 1,000,000 (쉼표 구분)
- **소수점**: 한국어는 소수점 아래 쉼표, 영어는 점 사용
- **통화**: ₩1,000 (한국어), $1,000 (영어)
