# EPUB 본문 텍스트 정제(Text Cleanup)

## 목적
PDF에서 추출된 텍스트에는 종종 머리말/꼬리말(예: 페이지 번호, 파일명, 변환기 워터마크)이 섞여 들어옵니다.
이 값들이 그대로 EPUB 본문(실제 책 내용)에 들어가면 읽기 경험이 크게 나빠집니다.

이 문서는 "EPUB 본문에 포함되지 않아야 하는 텍스트"를 규칙으로 정의하고,
코드에서 어떤 단계에서 어떻게 제거하는지(정제 단계) 기준을 제공합니다.

비유로 설명하면, PDF에서 텍스트를 뽑는 과정은 "책 내용을 사진으로 찍어 OCR로 글자를 읽는 것"과 비슷합니다.
이때 페이지 하단의 쪽수(페이지 번호)까지 같이 읽히는데, EPUB은 "쪽수가 없는 디지털 책"이므로 이런 값은 지워주는 것이 맞습니다.

## 제거 대상(기본 규칙)
아래 항목은 기본적으로 EPUB 본문에서 제거합니다.

- 페이지 표시 문구
  - 예: `페이지 25`
  - 예: `Page 25`
- 쪽수 형태의 꼬리말
  - 예: `- 25 -`
  - 예: `— 25 —`
- 위 항목과 함께 등장하는 "쪽수만 있는 줄"
  - 예: `25`
- 원본 파일명/경로로 보이는 줄
  - 예: `260215 주일말씀.hwp`
- 변환기/서비스 워터마크로 보이는 줄
  - 예: `PdfToEpub Converter`
  - 예: `PDF to EPUB Converter`
- 저작권 안내/배너로 보이는 줄
  - 예: `이 자료는 저작권에 의해 보호됩니다.`
  - 예: `다음에서 발췌`

## 예시
입력(원문 일부):

```
페이지 25
25
260215 주일말씀.hwp
- 25 -

(이후 본문...)
```

출력(정제 후):

```
(이후 본문...)
```

## 적용 지점(파이프라인)
정제는 "EPUB로 렌더링하기 직전"에 적용합니다.

- PDF 텍스트 기반:
  - `PDFExtractor`로 텍스트 추출
  - (옵션) 청크 경계 문맥 보정(LLM)
  - (옵션) 문서 흐름 재정리(LLM)
  - 정제 적용
  - XHTML 생성 → EPUB 생성

- 스캔(OCR/LLM) 기반:
  - OCR/LLM 결과(마크다운) 생성
  - (옵션) 문서 흐름 재정리(LLM)
  - 정제 적용
  - XHTML 생성 → EPUB 생성

## 설계 원칙
- 본문(실제 내용)과 무관한 "페이지 메타(부가 정보)"만 제거합니다.
- 숫자 한 줄처럼 애매한 값은 단독 규칙으로 과도하게 제거하지 않고, 페이지 표시 패턴과 인접한 경우에만 제거합니다.
- 규칙은 코드에서 한 곳에 모아 재사용 가능하게 유지합니다.
