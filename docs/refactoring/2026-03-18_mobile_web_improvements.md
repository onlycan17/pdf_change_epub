[배경]
- 현재 프론트엔드는 Tailwind 반응형 클래스가 일부 적용되어 있어 모바일에서 화면이 열리기는 하지만, 핵심 사용 흐름인 헤더 메뉴, 업로드, 변환 진행, 다운로드 화면이 작은 화면 기준으로 충분히 다듬어져 있지 않습니다.
- 특히 모바일에서는 화면 폭이 좁고 손가락으로 조작하므로, 데스크톱에서 괜찮았던 배치도 실제 사용 시 답답하거나 누르기 어렵게 느껴질 수 있습니다.

[목표]
- 모바일 웹에서도 주요 기능을 무리 없이 사용할 수 있도록 핵심 화면의 레이아웃과 터치 영역을 개선합니다.
- 헤더 메뉴를 작은 화면에서도 쉽게 열고 닫을 수 있게 정리합니다.
- 업로드, 변환 진행, 다운로드 화면에서 버튼과 정보 카드가 한 줄에 무리하게 눌리지 않도록 재배치합니다.

[비목표]
- 이번 작업에서 별도 모바일 앱(PWA, 앱스토어 배포)은 구현하지 않습니다.
- 모든 페이지를 전면 재디자인하지 않고, 실제 전환 흐름에 직접 연결되는 핵심 화면을 우선 개선합니다.

[구현 계획]
- 공통 레이아웃
  - 모바일 화면에서 상하 여백과 안전 영역(safe area)을 고려한 공통 여백 유틸리티를 추가합니다.
- 헤더
  - 모바일 메뉴 버튼 접근성을 높이고, 메뉴가 열렸을 때 현재 상태를 더 명확히 보여주도록 개선합니다.
  - 메뉴 오버레이와 닫기 버튼을 추가해 작은 화면에서도 탐색이 쉽도록 만듭니다.
- 업로드/진행/다운로드 페이지
  - 카드 내부 패딩, 버튼 너비, 안내 문구 줄바꿈을 모바일 우선으로 조정합니다.
  - 모바일에서 가장 중요한 행동 버튼은 더 넓고 누르기 쉬운 크기로 제공합니다.
  - 진행 상태 목록과 하단 링크 영역은 세로 배치 기준으로 자연스럽게 읽히도록 재구성합니다.

[영향 범위]
- 프론트엔드 레이아웃과 스타일이 변경됩니다.
- 백엔드 API와 데이터 구조에는 영향이 없습니다.

[검증 계획]
- `npm run format`
- `npm run lint`
- `npm run build`

[완료 기준]
- 390px 전후의 모바일 폭에서도 헤더 탐색, 파일 선택, 변환 진행 확인, 다운로드 버튼 클릭이 가로 스크롤 없이 가능해야 합니다.
- 버튼과 링크가 손가락으로 누르기 쉬운 크기로 유지되어야 합니다.
