# 애드센스 승인 대비 공개 콘텐츠 보강 계획

## 배경

- 애드센스 심사에서 `게시자 콘텐츠가 없는 화면에 Google 게재 광고`, `가치가 별로 없는 콘텐츠` 사유가 확인되었다.
- 현재 사이트는 도구형 서비스 중심 구조라서 로그인, 업로드, 변환 같은 작업 화면 비중이 크다.
- 첫 HTML 응답에 본문이 거의 없고, 일부 공개 페이지는 분량이 짧거나 `준비중` 상태가 남아 있어 심사 봇이 콘텐츠 부족으로 판단할 가능성이 있다.

## 목표

- 공개 방문자가 읽을 수 있는 원본 설명 콘텐츠를 늘린다.
- 광고성 요소를 공개 정보 화면보다 앞세우지 않도록 정리한다.
- 첫 HTML 응답에서도 서비스 설명과 주요 링크를 확인할 수 있게 한다.

## 비목표

- 실제 결제 기능이나 대용량 요청 백엔드 구조를 바꾸지 않는다.
- 인증 정책 자체를 전면 개편하지 않는다.
- 애드센스 승인 자체를 보장한다고 가정하지 않는다.

## 영향 범위

- 프론트엔드 공개 페이지 라우팅
- 홈 화면과 정보성 페이지의 본문 구성
- 초기 HTML의 기본 설명 콘텐츠
- 푸터 링크 및 `준비중` 문구 정리

## 변경 전략

1. 홈 화면에 서비스 작동 방식, 사용 대상, 변환 품질 기준, 공개 정책 정보를 추가한다.
2. 도움말, 문의, 약관, 개인정보, 대용량 요청, 후원 페이지를 공개 정보 페이지 중심으로 보강한다.
3. 공개 페이지에서는 광고 자리 표시자와 `준비중` 문구를 제거하거나 축소한다.
4. 로그인/업로드/변환 같은 도구 화면보다 정보성 페이지가 더 풍부하게 보이도록 공개 링크 구조를 정리한다.
5. `index.html`에 자바스크립트 실행 전에도 읽을 수 있는 기본 설명 블록을 넣어 초기 HTML 가독성을 높인다.

## 검증 계획

- `npm run format`
- `npm run lint`
- `npm run build`

검사 중 하나라도 실패하면 즉시 원인을 수정하고 전체 검사를 다시 실행한다.

## 롤백 전략

- 공개 페이지 텍스트 변경이 문제를 만들면 해당 페이지 단위로 되돌린다.
- 라우팅 변경이 예상과 다르면 공개 전용 설명 블록만 남기고 보호 라우팅을 복구한다.
