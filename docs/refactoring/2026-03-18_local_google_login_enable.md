[배경]
- 로컬 개발 환경에서 로그인/회원가입 화면에 Google 간편로그인 버튼이 표시되지 않고 있습니다.
- 현재 프론트엔드는 `VITE_GOOGLE_ALLOWED_ORIGINS` 값이 비어 있으면 개발 환경에서 Google 로그인을 막도록 구현되어 있습니다.
- 운영 서버의 Google 로그인 동작은 이미 사용 중이므로, 운영 설정이나 운영 허용 원본 정책은 건드리지 않아야 합니다.

[목표]
- 운영 환경 설정을 변경하지 않고, 로컬 개발 환경(`localhost`, `127.0.0.1`)에서만 Google 간편로그인이 표시되고 동작하도록 개선합니다.
- 명시적으로 허용 원본 목록이 설정된 경우에는 기존 설정을 그대로 우선 적용합니다.

[비목표]
- Google Cloud Console의 운영 OAuth 설정 자체를 바꾸지 않습니다.
- 운영 도메인 허용 원본 정책을 완화하지 않습니다.

[구현 계획]
- 프론트엔드 Google 로그인 버튼 컴포넌트에 개발 환경 전용 기본 허용 원본 목록을 추가합니다.
- `VITE_GOOGLE_ALLOWED_ORIGINS`가 비어 있을 때만 로컬 주소(`http://localhost:3000`, `http://127.0.0.1:3000`, `http://localhost:5173`, `http://127.0.0.1:5173`)를 자동 허용합니다.
- 환경 예시 문서에 로컬 개발 동작 방식을 함께 설명해 설정 의도를 문서와 코드에 맞춥니다.

[영향 범위]
- 프론트엔드 로그인/회원가입 화면의 Google 버튼 표시 조건이 변경됩니다.
- 백엔드 토큰 검증이나 운영 로그인 처리에는 영향이 없습니다.

[검증 계획]
- `npm run format`
- `npm run lint`
- `npm run build`
- 로컬 로그인 페이지에서 Google 버튼 표시 여부 확인

[완료 기준]
- 운영 환경에서는 기존 허용 원본 정책이 유지됩니다.
- 로컬 개발 환경에서는 별도 허용 원본 설정이 없어도 Google 버튼이 표시됩니다.
