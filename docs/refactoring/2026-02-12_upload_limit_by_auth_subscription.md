# 2026-02-12 업로드 용량 정책(비로그인/로그인/구독) 적용

## [배경]
- 현재 업로드 제한은 단일 최대치로만 동작해, 비로그인 사용자와 구독 사용자의 정책 차등이 반영되지 않습니다.
- 요구사항은 아래와 같습니다.
  - 비로그인: 최대 25MB
  - 로그인 + 구독 활성: 최대 300MB
  - 25MB 초과는 로그인/구독 필요

## [계획]
- 변환 시작 API(`/api/v1/conversion/start`)에서 요청별 정책을 계산해 서버에서 강제합니다.
  - Authorization Bearer 토큰의 클레임으로 로그인/구독 상태 판별
  - 구독 판단 클레임: `is_subscribed`, `subscription_active`, `plan`, `subscription_plan`
- 토큰 발급 API는 테스트/개발 편의를 위해 사용자 유형별 클레임을 부여합니다.
  - `premiumuser / testpass` 로그인 시 구독 활성 클레임 포함
- 프론트 업로드 화면 안내 문구/사전 검사도 정책과 동기화합니다.
- 통합 테스트를 추가해 정책 회귀를 방지합니다.

## [영향도]
- API 인터페이스는 동일하며, 초과 업로드 시 에러 메시지가 정책 기반으로 달라집니다.
- 서버에서 정책을 최종 강제하므로 클라이언트 우회 시도에도 제한이 유지됩니다.
