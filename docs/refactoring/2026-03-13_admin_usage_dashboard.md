[배경]
- 운영자 계정이 시스템 사용 현황을 한눈에 확인할 수 있는 대시보드가 필요합니다.
- 현재 코드에는 운영자 판별(`is_privileged`)과 대용량 요청 관리 화면은 있지만, 전체 운영 상황을 숫자와 최근 활동으로 모아 보여주는 화면은 없습니다.

[목표]
- 운영자 계정만 접근 가능한 사용 현황 대시보드를 추가합니다.
- 기존 데이터 원천(사용자 저장소, 무료 사용량 장부, 대용량 요청 장부, 현재 변환 작업 저장소)을 재사용합니다.
- 첫 버전은 “지금 바로 운영에 쓸 수 있는 요약판”에 집중합니다.

[비목표]
- 별도 분석 DB나 장기 보관용 이벤트 파이프라인은 이번 범위에 포함하지 않습니다.
- 일반 사용자용 통계 화면은 만들지 않습니다.

[구현 계획]
- 백엔드
  - 운영자 전용 대시보드 응답 모델 추가
  - 사용자 수, 최근 무료 사용량, 대용량 요청 상태, 현재 변환 작업 현황을 조합하는 서비스 추가
  - 운영자 계정만 접근 가능한 `/api/v1/admin/dashboard` 엔드포인트 추가
- 프론트엔드
  - 운영자 전용 대시보드 페이지 추가
  - 헤더와 라우터에 운영자 전용 진입점 추가
  - 핵심 숫자 카드, 최근 사용 추이, 최근 요청/변환 목록을 모바일 대응 레이아웃으로 구성
- 테스트
  - 운영자 접근 성공, 일반 사용자 접근 차단, 주요 집계값 응답 검증

[영향 범위]
- 인증: 운영자 계정 판별 로직을 재사용합니다.
- API: 운영자 전용 조회 엔드포인트가 추가됩니다.
- UI: 운영자 로그인 시 새로운 “운영대시보드” 진입점이 보입니다.

[검증 계획]
- 백엔드 테스트: 관리자 대시보드 API 테스트 추가
- 백엔드 린트/포맷: `ruff`, `black`
- 프론트엔드 포맷/린트/빌드: `npm run format`, `npm run lint`, `npm run build`

[롤백 전략]
- 새 엔드포인트와 프론트 페이지를 제거하면 기존 동작으로 즉시 되돌릴 수 있습니다.
