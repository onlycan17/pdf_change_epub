# 2026-02-11 EPUB 이미지 포함 및 문맥 보정 모델 폴백

## [배경]
- 현재 변환 파이프라인은 PDF 텍스트 중심으로 EPUB을 생성하며, PDF 내부 이미지는 EPUB 본문에 실제 파일로 포함되지 않습니다.
- 문맥 보정(청크 경계 연결)은 단일 모델 호출로 동작해, 모델 무응답/실패 시 복원력이 낮습니다.
- 요구사항: OpenRouter 기반으로 `deepseek/deepseek-v3.2`를 1차로 사용하고, 실패 시 `nvidia/nemotron-3-nano-30b-a3b`를 2차로 사용해야 합니다.

## [계획]
- EPUB 생성 서비스(`EpubGenerator`)에 이미지 리소스 입력 구조를 추가합니다.
- 오케스트레이터에서 PDF 이미지를 추출해 EPUB 리소스로 포함하고, 본문에 `<img>` 참조를 삽입합니다.
- 문맥 보정 서비스(`TextContextCorrector`)에 1차/2차 모델 폴백 로직을 추가합니다.
- OpenRouter API 키를 `OPENROUTER_API_KEY`에서도 읽도록 설정 경로를 보강합니다.
- 관련 단위 테스트를 보강해 폴백 호출과 이미지 전달을 검증합니다.

## [영향도]
- 변환 결과 EPUB에 이미지가 포함되어 원문 재현성이 향상됩니다.
- LLM 호출 안정성이 개선되어 문맥 보정 실패율이 낮아집니다.
- API 인터페이스 변경은 없고 내부 변환 품질/안정성만 개선됩니다.
