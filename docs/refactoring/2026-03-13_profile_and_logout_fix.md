[배경]
- 로그인 후 프로필 페이지가 실제 계정 정보를 보여주지 않고 임시 예시값을 표시하고 있습니다.
- 헤더의 로그아웃 버튼이 실제 로그아웃 처리와 연결되지 않아 인증 상태가 남아 있습니다.

[목표]
- 프로필 페이지가 현재 로그인한 사용자의 실제 계정 정보를 표시하도록 수정합니다.
- 헤더의 데스크톱/모바일 로그아웃 버튼이 토큰 삭제와 인증 상태 초기화를 정상 처리하도록 수정합니다.
- 로그인 직후 앱 상태와 사용자 정보가 일관되게 동기화되도록 보강합니다.

[비목표]
- 프로필 편집 저장 API나 변환 기록 영속 저장 기능은 이번 범위에 포함하지 않습니다.
- 백엔드 사용자 스키마를 크게 확장하지 않습니다.

[구현 계획]
- 프론트엔드 인증 유틸에 현재 사용자 조회, 토큰 기반 표시 이름 생성, 로그아웃 도우미를 추가합니다.
- 로그인/회원가입/구글 로그인 이후 앱 전역 상태에 사용자 정보를 반영합니다.
- 프로필 페이지는 실제 사용자 정보와 현재 구독 플랜을 기반으로 렌더링하도록 바꿉니다.
- 헤더의 로그아웃 버튼은 토큰 삭제, 사용자 상태 초기화, 로그인 페이지 이동을 수행하도록 연결합니다.

[영향 범위]
- 로그인 후 헤더와 프로필 페이지의 동작이 달라집니다.
- 인증 토큰이 없거나 만료된 경우 사용자 정보가 즉시 비워집니다.

[검증 계획]
- Playwright로 로그인 → 프로필 진입 → 로그아웃 흐름을 확인합니다.
- `npm run format`, `npm run lint`, `npm run build`를 실행합니다.
