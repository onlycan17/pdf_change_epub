# 회원가입 복구 및 Google 전용 간편인증 전환

## 배경

- 로그인 페이지에는 GitHub 간편인증 버튼이 있으나 실제 인증 연동은 구현되어 있지 않다.
- 회원가입 페이지는 폼만 있고 실제 API 호출이 없어, 사용자 입장에서는 가입 버튼이 동작하지 않는다.
- 사용자는 GitHub 간편인증을 제거하고 Google 간편인증만 유지하길 원한다.

## 목표

- GitHub 간편인증 UI를 제거한다.
- 이메일 기반 회원가입을 실제 API와 연결해 가입이 되도록 만든다.
- 기존 이메일 로그인과 Google 로그인은 유지한다.
- Playwright로 회원가입과 로그인 흐름을 실제 브라우저에서 검증한다.

## 비목표

- Google 로그인 공급자 자체를 새로 연동하지 않는다.
- OAuth 공급자를 추가하지 않는다.
- 인증 체계를 외부 DB 기반으로 전면 개편하지 않는다.

## 영향 범위

- 백엔드 인증 API와 사용자 저장소
- 프론트엔드 로그인/회원가입 페이지
- 인증 테스트

## 구현 전략

1. 사용자 저장소에 이메일/비밀번호 계정을 저장할 수 있도록 필드를 확장한다.
2. `/api/v1/auth/register` 엔드포인트를 추가하고, 가입 후 이메일 로그인으로 이어질 수 있게 한다.
3. `/api/v1/auth/token`에서 데모 계정과 실제 가입 계정을 함께 처리한다.
4. 프론트엔드 회원가입 폼을 실제 API 호출로 연결한다.
5. 로그인/회원가입 화면에서 GitHub 버튼을 제거하고 Google 버튼만 남긴다.
6. Playwright로 회원가입, 로그인, 실패 케이스를 점검한다.

## 검증 계획

- `python3 -m pytest tests/test_auth_api.py`
- `python3 -m ruff check app tests`
- `python3 -m mypy app tests`
- `npm run format`
- `npm run lint`
- `npm run build`
- Playwright 수동 시나리오 실행

## 롤백 전략

- 인증 저장소 필드 변경이 문제를 만들면 신규 컬럼을 사용하지 않는 기존 인증 흐름으로 되돌린다.
- 프론트 회원가입 연결이 실패하면 로그인 기능은 유지하고 회원가입 버튼을 임시 비활성화한다.
