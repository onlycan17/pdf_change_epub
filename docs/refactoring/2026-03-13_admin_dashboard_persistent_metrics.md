[배경]
- 현재 운영자 대시보드는 “서버가 지금 기억하는 최근 작업”은 보여주지만, 서버 재시작 후에도 남는 월간 추이와 실패 현황은 충분히 보여주지 못합니다.
- 운영 판단에는 일회성 메모보다 누적 장부가 더 중요하므로, 변환 작업의 최신 상태를 SQLite에 남기는 보강이 필요합니다.

[목표]
- 변환 작업의 최신 상태를 SQLite에 저장합니다.
- 운영자 대시보드에 최근 30일 변환 추이, 상태별 분포, 실패 작업 목록을 추가합니다.
- 기존 변환 동작은 유지하고, 통계만 보강합니다.

[비목표]
- 별도 분석 시스템이나 외부 데이터 웨어하우스는 도입하지 않습니다.
- 사용자별 정교한 코호트 분석(그룹별 장기 비교)은 이번 범위에서 제외합니다.

[구현 계획]
- 백엔드
  - 변환 작업 최신 상태를 저장하는 SQLite 스냅샷 테이블 추가
  - `ConversionJobStore`의 생성/수정/취소 흐름에서 스냅샷을 자동 반영
  - 최근 30일 일별 건수, 상태별 집계, 최근 실패 목록을 조회하는 서비스 추가
  - 운영자 대시보드 응답 모델과 서비스 확장
- 프론트엔드
  - 최근 30일 변환 추이와 실패 목록 섹션 추가
  - 상태 분포 요약 카드 추가
- 테스트
  - 스냅샷 저장 및 대시보드 집계 테스트 추가

[영향 범위]
- 변환 시작/업데이트/완료/실패 시 DB 쓰기가 한 번 더 발생합니다.
- 대시보드는 더 안정적인 누적 통계를 보여줄 수 있게 됩니다.

[검증 계획]
- 백엔드 테스트: 관리자 대시보드 테스트 확장
- 백엔드 린트/포맷: `ruff`, `black`
- 프론트엔드 포맷/린트/빌드: `npm run format`, `npm run lint`, `npm run build`
