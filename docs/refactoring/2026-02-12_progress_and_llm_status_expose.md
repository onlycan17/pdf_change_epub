# 2026-02-12 변환 60% 정체 체감 개선 및 LLM 상태 노출

## [배경]
- 변환 진행률이 60%에서 장시간 멈춘 것처럼 보여 사용자 입장에서 작업 중단으로 오해될 수 있습니다.
- 실제로는 문맥 보정(LLM 호출) 단계에서 처리 중일 수 있으나, 현재 UI/API에서 이를 충분히 설명하지 못합니다.

## [계획]
- 문맥 보정 단계(`context_correction`)에서 청크 단위 진행률을 세분화해 60% 이후에도 지속적으로 상승하도록 개선합니다.
- 상태 응답에 아래 LLM 메타데이터를 추가합니다.
  - `llm_used_model`: 실제 성공 모델
  - `llm_attempt_count`: 호출 시도 횟수
  - `llm_fallback_used`: 폴백 모델 사용 여부
- 프론트 변환 화면에서 해당 메타데이터를 노출합니다.

## [영향도]
- API 호환성: 필드 추가(하위 호환)
- 사용자 체감: 진행 중/정체 구분이 쉬워짐
- 운영 진단: 모델 폴백 여부를 상태 API로 즉시 확인 가능
