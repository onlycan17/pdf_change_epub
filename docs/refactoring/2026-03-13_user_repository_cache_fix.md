[배경]
- `tests/test_auth_api.py`를 다시 실행했을 때 회원가입 테스트가 `409 Conflict`로 실패했습니다.
- 원인을 확인한 결과 `UserRepository` 전역 캐시가 이전 SQLite 경로를 계속 재사용해, 새 테스트가 다른 임시 DB가 아니라 기존 DB를 읽는 문제가 있었습니다.

[계획]
- `get_user_repository()`가 현재 설정의 DB 경로와 캐시된 저장소의 DB 경로를 비교하도록 변경합니다.
- `TestingSettings`가 `DB_URL` 환경변수를 무시하고 항상 `./test.db`를 쓰는 문제를 함께 수정합니다.
- 경로가 다르면 새 `UserRepository`를 생성하도록 수정해 테스트 격리를 보장합니다.
- 변경 후 인증 테스트와 프론트엔드/백엔드 자동 검사를 다시 실행합니다.

[영향도]
- 기능 동작은 유지되며, 저장소 캐시가 잘못된 DB를 재사용하던 문제만 수정됩니다.
- 로컬 회원가입, 로그인, 테스트 환경 분리에 직접 영향을 줍니다.
