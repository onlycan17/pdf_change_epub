[배경]
- 운영자 대시보드에 최근 실패 목록은 추가됐지만, 실패가 어떤 유형으로 몰리는지는 한눈에 파악하기 어렵습니다.
- 운영자는 “실패 건수”보다 “무슨 종류의 실패가 반복되는지”를 알아야 대응 우선순위를 정할 수 있습니다.

[목표]
- 실패 작업을 오류 메시지 기준으로 분류해 대시보드에 집계판을 추가합니다.
- 분류 기준은 현재 저장 중인 `error_message`와 `current_step`만 사용합니다.
- 구독 전환율처럼 근거가 약한 지표는 이번 단계에 포함하지 않습니다.

[구현 계획]
- 백엔드
  - 실패 작업의 오류 메시지를 카테고리로 분류하는 규칙 추가
  - 최근 실패 작업 장부에서 카테고리별 건수를 집계
  - 운영자 대시보드 응답에 실패 원인 분포 추가
- 프론트엔드
  - 실패 원인 분포 카드/막대 UI 추가
  - 최근 실패 목록과 함께 읽기 쉬운 장애 분석 영역으로 정리
- 테스트
  - OCR/파일/외부 서비스/기타 분류가 기대대로 집계되는지 검증

[영향 범위]
- 변환 동작 자체는 바뀌지 않습니다.
- 운영자 대시보드 해석력(문제 파악 속도)만 좋아집니다.

[검증 계획]
- 백엔드 테스트 확장
- `ruff`, `black`, `pytest`
- `npm run format`, `npm run lint`, `npm run build`
