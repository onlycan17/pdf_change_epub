#+ 무료 운영(광고/후원) 전환 메모

## 배경
- 현재 개인사업자번호 부재로 결제(카드/구독) 기능을 당장 붙이기 어렵습니다.
- 우선 무료 기능만 제공해 시장 반응(사용자 수, 재방문, 전환 니즈)을 확인합니다.

## 현재 운영 정책
- 결제/구독 기능은 비활성화합니다.
- 업로드/변환 제한은 모두 무료 기준으로 고정합니다.
- 수익화는 2가지로 시작합니다.
  - 광고: UI에 광고 영역(슬롯)을 배치하고, 실제 광고 SDK는 추후 연결합니다.
  - 후원: 서비스 유지 비용을 안내하고 후원 계좌 정보를 노출합니다.

## 제품(UI) 변경 사항
- 상단/하단 내비게이션에서 `프리미엄` 항목을 `후원`으로 교체합니다.
- 기존 `/premium` 및 결제 성공/실패 페이지는 사용자 혼선을 줄이기 위해 `후원` 안내로 연결합니다.

## 백엔드 변경 사항
- `/api/v1/billing/*`는 호환성을 위해 라우트 자체는 유지하되, 기본적으로 `503`(일시 중단)로 응답합니다.
- `GET /api/v1/billing/plans`는 UI가 참고할 수 있도록 `free` 플랜만 반환합니다.

## 결제 기능 재도입(추후)
- 환경 변수 플래그로 결제 기능을 다시 활성화할 수 있게 설계합니다.
  - 예: `APP_BILLING_ENABLED=true`
- 프론트엔드도 동일하게 플래그(`VITE_BILLING_ENABLED=true`)를 기준으로 프리미엄/결제 UI를 노출합니다.
