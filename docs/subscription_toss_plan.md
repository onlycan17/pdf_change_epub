# 토스페이먼츠 구독 결제 연동 구현 문서(요청 반영용)

## 1) 목표
- 월 구독(`monthly`)과 연 구독(`yearly`) 요금제 제공
- 연 구독은 월 구독 대비 10% 할인 적용
- 구독 플랜별 업로드 용량 차등 적용
- 기존 엔드포인트(`/api/v1/billing/*`)와 최대한 호환된 형태로 전환

## 2) 결정한 정책
- 무료 플랜: `free`
  - 업로드 상한: `25MB`
- 월 구독: `monthly`
  - 업로드 상한: `300MB`
  - 월 결제 금액: `9900원`
- 연 구독: `yearly`
  - 업로드 상한: `500MB`
  - 연간 결제 기본 금액: `12 × 9900원 × (1 - 10%) = 106,920원`

## 3) 백엔드 연동 설계
1. `app/services/subscription_plans.py`
   - 플랜 코드 정규화(`monthly`, `yearly`, `free`)
   - 플랜별 업로드 한도/가격(원)/할인율 노출
2. `app/api/v1/billing.py`
   - 기존 라우트 유지
   - 새 라우트 추가: `GET /api/v1/billing/plans`
3. `app/services/stripe_service.py`
   - 클래스명은 기존 코드 의존성을 유지하되 내부 구현을 토스 결제 API 기반으로 변경
   - `create_checkout_session`에서 월/연 플랜을 받아 토스 결제 요청 생성
   - 연간 할인 정보는 플랜 정책 기반으로 계산
   - `get_stripe_service` 의존성은 유지

## 4) 변환 한도 반영 지점
- `app/api/v1/conversion.py`
  - JWT 토큰 payload에서 플랜 판별 후 업로드 제한을 플랜별 한도로 계산
  - 익명/무료/연간 구독별 메시지 분기

## 5) 프론트 반영 방식
- `/premium`
  - 백엔드 플랜 목록 API로 가격/할인/제한 정보 조회
  - 월간/연간 카드 버튼 클릭 시 checkout 세션 생성 후 리다이렉트
- `/upload`
  - 토큰에서 플랜을 판별해 업로드 크기 메시지/체크 기준 반영
- `/profile`
  - 현재 플랜 라벨과 업로드 용량 한도 표시

## 6) 테스트 반영
- Billing API 테스트: 플랜 목록 조회 케이스 추가
- 변환 API 제한 테스트:
  - 익명/무료/월간/연간 구독별 한도 분기 확인
